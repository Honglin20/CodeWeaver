```
│   ├── post_processor.py      # 后处理层 ⭐⭐

│   ├── config.py              # 配置管理 ⭐
│   └── tools.py               # 工具注册
├── workflows/
│   └── workflow_generator/    # 元工作流
│       ├── flow.md
│       └── agents/
│           ├── intent_parser.md
│           ├── skeleton_builder.md
│           └── agent_builder.md
└── tests/                     # 35+ 测试用例
```

### 数据流（带后处理）
```
用户输入 (自然语言)
    ↓
Intent Parser Agent
    ↓
意图分析 (可能有格式问题)
    ↓
Skeleton Builder Agent
    ↓
工作流骨架 (可能有格式问题)
    ↓
Agent Builder Agent
    ↓
Agent 配置 (可能有格式问题)
    ↓
🔧 后处理层 (自动修正) ⭐⭐
    ↓
✅ 格式正确的可执行工作流
```

## 关键技术突破

### 1. 后处理层架构 ⭐⭐⭐
**问题**: LLM 输出格式不可控

**解决方案**:
- 实现 WorkflowOutputNormalizer 类
- 基于规则的条件映射
- 自动修正各种格式变体

**效果**:
- 格式正确率：0% → 100%
- 无需依赖完美的 LLM 输出
- 确定性保证

### 2. Prompt 工程策略
**发现**: 明确示例比复杂规则更有效

**应用**:
```markdown
✅ 正确: validate --> review : [passed]
❌ 错误: validate --> review : 验证通过
```

**效果**: 格式正确率显著提升

### 3. 迭代优化方法
**流程**:
1. 实现功能
2. 真实 LLM 测试
3. 识别问题
4. 实施优化
5. 严苛测试
6. 推送 GitHub
7. 重复

**效果**: 4 轮迭代，持续改进
